  • Question 1: What are the difference between var, let and const ?

    Answer:
    var let const
    1. var can be re-declared and updated. let can be updated but not re-declared. const neither can be re-declared nor updated.
    2. var declarations are globally scoped or function scoped. let declarations are block scoped. const declarations are block scoped as like as let.
    3. var can be declared without initialized. let can be declared without initialized. const must be initialized during declaration.
    4. var can be accessed without initialization as its default value is “undefined”. let can't be accessed without initialization. const can't be accessed without initialization.

  • Question 2: What are the difference between arrow function and regular function ?

    Answer:
    arrow function regular function
    1. const num = (a, b) => a + b; function num(a, b) { return a + b;}
    2. If arrow function, contains one expression, curly braces {} can be omitted and the expression will be implicitly returned. In regular function, return keyword have to be used to return any value, if nothing will be returned, the it will return undefined.
    3. arrow function don't have it's own this. regular function have it's own this.
    4. In arrow function, duplicate parameter can't be used. In regular function, duplicate parameter can be used.

  • Question 3: What are the difference between map(), forEach(), filter() and find() ?

    Answer:
    map ( ) forEach ( ) filter ( ) find ( )
    1. map () returns a newArray that is created based on callback function. forEach () returns value as undefined. filter () returns all element of an array what are matched with condition. find () returns only first element of an array what is matched with condition.
    2. map (), doesn't affect on the original array. forEach(), doesn't affect on the original array. filter (), returns an array. find (), returns only number.
    3. map (), works with operation. forEach(), also works with operation, same as map (). filter (), works with condition. find (), works with condition like as filter ().
